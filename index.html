01 학술대회 · 연구실

OFF Board Charger의 연속적인 충전 동작을
보장하기 위한 모듈 제어 로직

2023 · 인천대학교 전력변환연구실

왜 했나

300kW 급속충전기는 30kW 모듈 10개로 구성되는데, 저전압↔고전압 전환 시 모든 모듈이 동시에 릴레이를 변환하면 짧은 충전 공백이 생긴다. 이 공백은 충전 소자에 전기적 스트레스를 주고 안정성을 떨어뜨린다. 이 문제를 없애고 싶었다.

어떻게 구상했나

모든 모듈을 동시에 전환하는 대신, 하나씩 순차적으로 전환하는 제어 로직을 설계했다. 전환 중에도 나머지 모듈이 충전을 이어받아 공백 없이 연속 충전이 가능하고, 급격한 출력 변동도 없애 소자 수명까지 챙겼다.

02 자율주행 · ROS2 · OpenCV · LiDAR

LIMO 자율주행
경진대회

2024. 08 · 인천대학교

어떤 프로젝트인가

카메라와 LiDAR를 이용해 지그재그, 급곡선, 장애물 회피, 라바콘 슬라롬, 색깔 차선 인식, 좁은 통로 통과까지 6가지 미션을 수행하는 자율주행 알고리즘 경진대회. WeGo LIMO 플랫폼(Jetson Nano)에서 단독으로 전체 주행 로직을 구현했다.

어떤 방식들을 고려했나

차선 인식은 딥러닝 기반 세그멘테이션도 고려했지만 Jetson Nano 연산 부하와 실시간성을 고려해 HLS 색공간 기반 방식을 선택했다. 장애물은 카메라로 감지하는 방법도 있었지만 라바콘처럼 형태가 다양한 경우에도 안정적으로 대응하려면 거리 측정이 직접 되는 LiDAR가 더 적합하다고 판단했다.

어떻게 구상했나

차선은 HLS 색공간에서 흰색과 노란색을 동시에 감지하고, 검출된 픽셀들의 무게중심 x좌표를 기준점과 비교해 조향값을 계산했다. 조명 변화에 강하게 threshold를 넉넉하게 잡았다. 장애물과 라바콘은 동일하게 LiDAR raw scan으로 0.5m 이내 점을 카운팅하고 어느 방향에 더 몰려있는지 각도까지 계산해 좌/우 회피 방향을 결정했다. 이 둘을 우선순위 기반으로 통합했다 — LiDAR 경고 → 차선 추종 순서로.

03 임베디드 · CAN · MQTT · AI

군집주행 Platooning

2025. 12 · SSAFY 임베디드 프로젝트

어떤 프로젝트인가

선두 차량은 사람이 수동으로 조작하고, 뒤따르는 차량들은 앞차의 속도와 방향을 실시간으로 받아 자동으로 추종하는 3대 군집주행 시스템. 선두가 움직이면 나머지가 알아서 따라온다.

왜 했나

단순히 코드만 짜는 게 아니라, 하드웨어 레이어(CAN)부터 네트워크 레이어(MQTT)까지 end-to-end로 직접 설계해보고 싶었다. 실제 차량 제어에서 통신 지연이 얼마나 치명적인지 몸으로 느끼고 싶었던 것도 있다.

어떻게 구상했나

차량 내부는 실시간성이 중요하니 CAN(500kbps)으로, 차량 간은 WiFi MQTT로 이중 통신 구조를 설계했다. ESP32가 CAN↔MQTT 게이트웨이 역할을 한다. 앞차 인식은 RPi4에서 TFLite 온디바이스 추론으로 통신 지연 없이 로컬에서 조향까지 완결되도록 했다. 사람이 앞에 있을 때 돌진하는 사고를 막으려고 클래스별 threshold도 다르게 잡았다 — car는 0.9, saram은 0.5.

배운 것

1회 영상 촬영 성공했지만 안정적 재현은 못 했다. WiFi MQTT 지연 + STM32 blocking 구조가 복합적으로 작용한 것으로 사후 분석했다. 분산 시스템의 불안정성은 단일 원인이 아니라 여러 레이어가 동시에 터지는 경우가 많다는 걸 직접 겪었다.

04 자율주행 · ROS2 · YOLO · AI

No-ill (노일)
독거노인 케어 로봇

2026. 01~02 · SSAFY 팀 프로젝트

어떤 프로젝트인가

독거노인이 혼자 있을 때 낙상이나 위험 상황이 생겨도 아무도 모르는 문제를 해결하려 했다. 로봇이 집 안을 순찰하다 사람을 발견하면 따라다니고, 낙상이 감지되면 보호자에게 자동 신고한다. STT/TTS 기반 대화 기능도 탑재해 노인과 직접 소통도 가능하다.

왜 했나

고령화 문제에서 실질적인 임팩트가 있는 걸 만들고 싶었다. 임베디드 + 자율주행 기술을 실생활 문제에 직접 붙여보는 경험이기도 했다.

어떻게 구상했나

두 가지 모드를 설계했다. 사람이 보이면 YOLO bbox로 직접 cmd_vel을 계산해 따라다니는 추적 모드, 사람이 안 보이면 Nav2 웨이포인트로 집 안을 순찰하는 순찰 모드. 추적은 Nav2를 쓰지 않았는데 사람이 움직일 때마다 goal을 재설정해야 해 반응이 느려지기 때문이었다. 낙상은 YOLO 2-class 모델(lying/other)로 감지하고, 지속되면 TTS로 확인 후 무응답 시 보호자에게 문자+현장 사진을 전송한다.

05 게임 · Unity · Photon Fusion 2

프로젝트 마네퀸

2026. 02~04 · SSAFY 팀 프로젝트

어떤 게임인가

마네킹 1명이 퇴마사 5명을 잡는 비대칭 호러 게임. 데드바이데이라이트 구조에 롤 칼바람나락식 랜덤 증강 요소를 더해, 매판 다른 조합으로 전략이 바뀐다.

왜 만들었나

숏츠·릴스처럼 콘텐츠 소비 단위가 짧아지는 트렌드에 주목했다. 증강·칼바람 같은 유행 요소를 게임에 접목해, 짧은 플레이타임 안에 매판 다른 플레이 경험을 주고 싶었다. 소규모 멀티플레이어라 전용 서버 없이 Photon Fusion 2를 선택했다.

어떤 걸 구현했나

퇴마사 클라이언트 전체를 담당했다. 카메라 벽 뚫림 방지(SphereCast), 애니메이션-타이머 불일치 해결(Animation Event 기반 이동잠금), 입력 권한 충돌 차단(HasInputAuthority 가드), Normal → Injured → Downed → Mannequin → Dead 생존 상태머신, 8종 랜덤 캐릭터 모델 네트워크 동기화까지 구현했다.

06 게임 · Unreal Engine 5.5 · C++20

인형뽑기 가게 시뮬레이터
Claw Machine Simulator

2026. 04~06 · SSAFY 팀 프로젝트

어떤 게임인가

인형뽑기 매장을 운영하는 1인칭 경영 시뮬레이터. 박스를 발주해 재고를 채우고, 인형을 진열하고, 손님을 받아 하루를 굴리며 매장을 키운다. Win64 / Steam 출시 싱글플레이 게임이고 Enhanced Input · BehaviorTree · NavMesh · DataTable 위에서 동작한다.

무엇을 맡았나

팀장으로서 전체 프로젝트의 진행 상황을 파악하며 팀원들에게 임무를 분배했다. 개발에서는 손님 NPC AI와 애니메이션, 그리고 튜토리얼을 담당했다.

어떻게 설계했나

기존 경영 시뮬레이터 게임들처럼 하루 단위로 게임이 흘러가게 만들고, 각 페이즈마다 플레이어가 직관적으로 해야 할 것 같은 임무들을 자연스럽게 배치했다.
