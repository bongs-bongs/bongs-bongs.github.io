01

전체 구조 — twist_mux 우선순위 체계

로봇이 사람을 추적하면서 동시에 장애물도 피하고, 대화 중엔 멈춰야 한다. 이 명령들이 충돌할 때 어떤 걸 우선할지를 twist_mux에 우선순위를 명시적으로 정의해서 해결했다. 각 노드는 독립적으로 자기 토픽에만 publish하고, twist_mux가 가장 높은 우선순위 명령을 선택해 실제 /cmd_vel로 내보낸다.

  우선순위 (높음 → 낮음)

  chat_stop_gate      (300) /cmd_vel_is_chatting  ← 대화 중: 무조건 정지
  safety_override     (200) /cmd_vel_safety        ← LiDAR 장애물: 즉시 회피
  person_override     (50)  /cmd_vel_person        ← 사람 추적/낙상 접근
  Nav2 (순찰)         (10)  /cmd_vel_nav           ← 웨이포인트 순찰
                               │
                          twist_mux
                               │
                           /cmd_vel
                               │
                          pca_drive_node
                               │
                     PCA9685 (I2C) → 모터 + 서보

안전에 관련된 노드일수록 숫자가 높다. 어떤 상황에서도 대화 중 정지 → 장애물 회피 → 사람 추적 → 순찰 순서가 깨지지 않는다.

02

PCA9685 I2C 드라이버 — 직접 구현

ROS2에서 모터·서보를 직접 제어하는 기성 드라이버가 맞지 않아서, PCA9685 PWM 칩을 I2C로 직접 제어하는 C++ 클래스를 만들었다. 파이썬 레퍼런스 코드를 보고 채널 매핑과 PWM 계산을 C++로 재구현했다.

pca_drive_node.cpp
// prescale = round(25MHz / (4096 × freq) - 1)
void set_freq_hz(int freq_hz) {
    int prescale = static_cast<int>(25000000.0 / (4096.0 * freq_hz) - 1.0 + 0.5);
    prescale = std::clamp(prescale, 3, 255);
    write_reg(0x00, 0x10); // MODE1 sleep
    write_reg(0xFE, prescale); // PRESCALE
    write_reg(0x00, 0xA0); // restart + AI
}

// cmd_vel linear.x → 스로틀 [-1..1] → PWM 변환, 가속 램프 적용
void tick() {
    double throttle = std::clamp(target_lin_ / max_lin_, -1.0, 1.0);
    if (std::fabs(throttle) < deadband_) throttle = 0.0; // 데드밴드

    // 가속 램프: 급출발 방지
    double diff = std::clamp(throttle - cur_throttle_, -accel_rate_ * dt, accel_rate_ * dt);
    cur_throttle_ += diff;

    motor_->set_throttle(cur_throttle_);
    servo_->set_from_angular_z(target_ang_, max_ang_);
}

서보 각도 변환도 직접 구현했다. angular.z를 받아 중심각 기준으로 좌/우 스팬을 비대칭하게 매핑해서 실제 조향각의 비선형 특성을 보정했다. 명령이 0.3초 이상 오지 않으면 자동으로 정지하는 timeout도 넣었다.

03

사람 추적 + 낙상 대응 — 두 모드 전환

person_override_node는 두 가지 모드를 가진다. 일반 추적(사람을 따라다니며 거리 유지)과 낙상 대응(낙상 감지 시 즉시 접근해서 도착 신호 발행). 낙상이 감지되면 추적보다 빠른 로직으로 덮어쓴다.

  /object_type (YOLO) ──── "lying" 또는 "others"
  /person_x (카메라 X좌표) ──── 시각 추종 제어
  /check_accident (낙상 판정) ──▶ fall_detected = true
  /scan (LiDAR) ──── 전방 거리 측정

         ┌──────────────────────────────────┐
  타이머  │ fall_detected?                   │
  20Hz  │   YES → handleFallAccident()     │
         │   NO  → handleNormalTracking()  │
         └──────────────────────────────────┘
person_override_node.cpp
// 낙상 접근: LiDAR 도착 판정 우선, 시간 기반 백업
void handleFallAccident() {
    // 1. LiDAR 거리 기반 도착 판정 (우선)
    if (front_distance_ <= arrival_dist) {
        triggerArrival("LiDAR distance");
        return;
    }
    // 2. 시간 기반 백업 (LiDAR 오탐 방어)
    if (elapsed >= fall_time) {
        triggerArrival("time-based fallback");
        return;
    }
    // 3. 시각 추적으로 접근 (카메라 X 좌표 비례 제어)
    performFallApproach();
}

// 일반 추적: 1m 이내면 정지, deadtime 초과하면 Nav2로 반환
void handleNormalTracking() {
    if (front_distance_ <= follow_stop_dist) {
        follow_stopped_ = true;
        publishStop();
        return;
    }
    if (elapsed > deadtime) return; // 명령 안 보냄 → twist_mux가 Nav2 선택
    performVisualTracking();       // 카메라 X 좌표 비례 제어
}

추적이 끊기면 명령을 아예 보내지 않는다. twist_mux가 타임아웃 처리로 자동으로 하위 우선순위(Nav2 순찰)로 전환되기 때문이다. 명시적 모드 전환 없이 우선순위만으로 자연스러운 추적→순찰 전환이 구현된다.

04

LiDAR 3섹터 장애물 회피 — 거리 비례 감속

단순히 "막히면 멈춤"이 아니라, 거리에 비례해서 속도를 줄이고 더 열린 방향으로 방향을 트는 능동 회피다. 전방(±15°)·좌측·우측 3개 섹터를 독립 측정해서 어느 쪽이 더 열려있는지 판단한다.

safety_override_node.cpp
// > 50cm: 개입 없음 (명령 미발행 → twist_mux가 하위 선택)
if (front_dist > slow_dist) return;

if (front_dist <= reverse_dist) {
    // < 30cm: 후진 + 반대 방향 (다음 전진 시 열린 공간 확보)
    cmd.linear.x = rev_speed;
    cmd.angular.z = -chooseAvoidanceYaw(left_dist, right_dist, yaw_reverse);
} else {
    // 30~50cm: 거리 비례 감속 + 회피 조향
    double t = (front_dist - reverse_dist) / (slow_dist - reverse_dist); // 0~1
    cmd.linear.x  = min_fwd * t;                  // 가까울수록 느리게
    cmd.angular.z = chooseAvoidanceYaw(...) * (1.0 - t); // 가까울수록 강하게
}

// 더 열린 쪽으로 회전
double chooseAvoidanceYaw(double left, double right, double magnitude) {
    if (left > right) return +magnitude;  // 좌측이 더 열림 → 좌회전
    return -magnitude;                    // 우측이 더 열림 → 우회전
}
05

순찰 모드 — Nav2 액션 기반 웨이포인트 순환

사람을 감지하지 못한 동안 로봇은 두 웨이포인트 사이를 반복 순찰한다. Nav2의 NavigateToPose 액션으로 목표를 보내고, 결과 콜백에서 다음 웨이포인트로 순환하는 구조다. 순찰 중 사람이 감지되면 즉시 현재 Goal을 취소하고 person_override에게 제어권을 넘긴다.

patrol_node.cpp
// YOLO에서 사람 감지 → 즉시 Nav2 Goal 취소
void objectTypeCallback(...) {
    const bool is_person = (type == "lying" || type == "others");
    if (is_person && !person_detected_) {
        person_detected_ = true;
        nav_client_->async_cancel_all_goals(); // 즉시 취소
    }
}

// Goal 완료 → 다음 웨이포인트로 순환
void goalResultCallback(...) {
    if (result.code == SUCCEEDED)
        current_waypoint_ = (current_waypoint_ + 1) % waypoints_.size();
    navigating_ = false; // 다음 tick에서 sendNextGoal() 자동 호출
}

// person_timeout(3초) 동안 미감지 시 순찰 재개
if ((now() - last_person_time_).seconds() > timeout)
    person_detected_ = false;

결과 · 배운 점

  • 사람 추적 → 순찰 → 낙상 대응 전환이 명시적 모드 전환 없이 twist_mux 우선순위만으로 자연스럽게 동작.
  • 각 노드가 독립적으로 동작하고 토픽으로만 통신하기 때문에, 한 노드를 재시작해도 전체 시스템이 중단되지 않음.
  • 낙상 도착 판정에서 LiDAR 단독이 아닌 "LiDAR 우선 + 시간 백업" 이중 판정 — 센서 단독 의존의 위험성을 시스템 레벨에서 방어.
  • PCA9685를 직접 I2C로 다루면서 PWM 레지스터 구조와 prescale 계산을 이해 — HAL 없이 하드웨어를 직접 제어하는 경험.
  • twist_mux 패턴: 여러 소스의 cmd_vel을 중재하는 구조는 실제 로봇 시스템에서 자주 쓰이는 설계임을 체감.
